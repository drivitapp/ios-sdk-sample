Other methods
  • settings()

    Returns the status of the framework. This class will give you the status of the framework and some permissions needed by the framework.

  • startTrip()

    This method will force a start of a Trip and will start saving location and activity of the user.

  • stopTrip()

    This method will force the SDK to stop a running trip if this exists, if a trip is not running the method will not do anything.

  • runningTrip()

    Call this method to get the trip that is running at the moment.

  • delegate(_:)

    To setup a delegate to listen to events given by the SDK. This events are for example: Trip did start, Trip did stop

  • pastTrips()

    Get a list of last trips that were registered, sorted by Date.

  • hasFilesToSinc(TripGUID:)

    This method will say if there is any files needed to upload with the metadata of this trip.

  • forceSyncOfTrip(TripGUID:)

    Force the sync of the files that still need to be uploaded to the server, if the trip has no files to be synced this method will not do anything.

  • checkIfUserHasTripsOrAccelToSync()
  • syncAccelAndTrips()

    Force the sync of trips metadata that were edited and need to be uploaded to the server.

  • sendUser()

    Force the sync of user settings that were edited.

  • userEditor()

    Method that will get the user settings and a class that will enabble the edition of those settings.

  • handleEventsForBackgroundURLSession(withIdentifier:completion:)

    Required method to be added to the appDelegate. This method will inform the SDK when an event happens with the sync of the files.

    • SessionIdentifier: The URLSession identifier.
    • completion: the completion handler to tell the system we are done.
  • applicationDidWake(withOptions:)

    Required method to be added to the appDelegate. This method will inform the SDK when an event happens in the regions.

    • options: The URLSession identifier.
    • completion: the completion handler to tell the system we are done.
